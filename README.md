# 우리사이

> 카카오톡 대화를 분석해 연인 사이의 온도를 알려주는 웹 서비스

**타겟:** "걔가 날 좋아하긴 하는 건가" 불안한 20대 연인

**배포:** https://t-gates.github.io/uri-sai/

---

## 현재 단계 — 수요 검증

실제 파싱 기능 없이 **자극적인 목업 통계**로 후킹 → 출시 알림(전화번호) 수집 → 전환율로 수요 판단.

### 퍼널 가설

| 단계 | 측정 지표 | 전략 |
|------|-----------|------|
| ① 유입 | 메타광고 CTR | 크리에이티브 A/B/C × feed/reels, 총 6장 |
| ② 전환 ⭐ | 랜딩 → 전화번호 신청 전환율 | 목업 통계를 최대한 자극적으로 — 이 숫자가 내 대화라면? |

목업만 보고도 전화번호를 남길 정도면 수요 있다고 판단.

## 핵심 전략

**프라이버시 바이 아키텍처** — txt 파일이 서버에 절대 안 감. 브라우저에서 전부 처리.

**자극 지표 후킹** — LLM 없이 통계만으로 "헉" 포인트 생성. 구체적 숫자가 추상적 약속보다 전환율 높음.

**무료 공유 카드 바이럴** — 인스타 스토리 사이즈 카드 + "나도 해보기" 링크 → 공유 루프.

## 아키텍처

```
React + Vite (SPA)
├── 배포: GitHub Pages (gh-pages 브랜치)
├── 트래킹: Meta Pixel + PostHog
├── 데이터: Supabase (출시 알림 전화번호)
└── UTM ?utm_content=secure → 프라이버시 라인 노출
```

### 트래킹 이벤트

| 이벤트 | 설명 |
|--------|------|
| `page_view` | 페이지 로드 |
| `modal_open` | CTA 버튼 클릭 |
| `phone_focus` | 전화번호 입력창 클릭 |
| `phone_submit` | 신청 완료 → Meta `Lead` 이벤트 동시 발사 |

> UTM 없이 접속한 유저는 트래킹 생략 (organic 방문자 제외)
